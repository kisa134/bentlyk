# Поселить Бентлика на постоянный сервер AWS

После перехода на WaveSpeed (мозг — удалённо) и Supabase (память — удалённо)
worker'у **не нужен GPU**. Это лёгкий Python-процесс, который крутится 24/7 и
дёргает API. Подойдёт самый дешёвый инстанс — `t3.micro` входит в **free tier**
(бесплатно первый год), дальше ~$8/мес.

Всё ставится само из одного скрипта. Ручной работы на сервере — ноль: ты только
создаёшь машину и вставляешь скрипт в поле «User data».

## Шаги

1. **Подготовь скрипт.** Открой `scripts/aws-bootstrap.sh`, заполни блок
   `SECRETS / CONFIG` сверху (обязательны `BENTLYK_LLM_API_KEY` — ключ WaveSpeed
   `wsk_...`, и `TELEGRAM_BOT_TOKEN`). Остальное — по желанию.

2. **Создай инстанс.** AWS Console → **EC2** → **Launch instance**:
   - **AMI:** Ubuntu Server 24.04 LTS
   - **Type:** `t3.micro` (free tier eligible)
   - **Key pair:** создай и скачай (на случай если захочешь зайти руками — но не
     обязательно)
   - **Network / Security group:** входящие порты **не нужны** (Бентлик только
     исходящие звонит). Можешь оставить дефолт или закрыть всё, кроме SSH со
     своего IP.
   - **Storage:** 10–20 GB хватит.

3. **Вставь скрипт.** Разверни **Advanced details** → найди поле **User data** →
   вставь туда **весь** заполненный `scripts/aws-bootstrap.sh`.

4. **Launch instance.** Подожди 3–4 минуты — машина сама ставит Docker, тянет
   репо, поднимает Бентлика и включает ночное автообновление.

## Проверить, что живёт

- **Дашборд** (как и раньше, на Vercel): `https://<твой>.vercel.app/api/dashboard?key=...`
  — баннер «ЖИВ», тело покажет инстанс AWS, сигналы дышат.
- **Telegram:** напиши боту — ответит. Скоро начнёт сам инициировать.
- **Логи на сервере** (если зайдёшь по SSH):
  ```
  tail -f /var/log/bentlyk-boot.log          # лог установки
  cd /opt/bentlyk && docker compose logs -f  # живой поток сознания worker'а
  ```

## Управление (по SSH, если нужно)

```
cd /opt/bentlyk
docker compose ps              # статус
docker compose logs -f         # логи
docker compose restart         # перезапуск
docker compose down            # остановить
sudo bentlyk-update            # вручную подтянуть новый код + пересобрать
```

Код обновляется сам каждую ночь в 04:30 (systemd timer `bentlyk-update.timer`).
Чтобы переключить источник на ветку `main` после мёрджа — поменяй `BRANCH=main` в
скрипте (или в `/opt/bentlyk` сделай `git checkout main`).

## Безопасность

- `.env` на сервере лежит с правами `600`. User-data в EC2 виден тем, у кого есть
  доступ к аккаунту/инстансу — держи аккаунт под 2FA и **ротируй ключи**, которые
  ты раньше светил в чате.
- `BENTLYK_ALLOW_CODE=1` включает запуск кода в песочнице **внутри контейнера**
  (изолирован от хоста). Это осознанно: это его собственное тело, он учится
  программировать и воплощать свои цели. Хочешь строже — поставь `=0`.
