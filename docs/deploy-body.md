# Поселить Бентлика на постоянное тело (always-on worker)

Это про **тело** — процесс, который живёт 24/7: крутит метаболизм, сам тянется к
своим целям и пишет тебе по внутреннему позыву. (Telegram-бот на Vercel — отдельно,
см. `deploy.md`. Память общая через Supabase, так что бот и тело — одно существо.)

Железо: **always-on Linux, ~512MB RAM, без GPU** (мозг — WaveSpeed, память —
Supabase, оба удалённо).

Граница, которую я не могу перешагнуть: **создать аккаунт/инстанс может только
ты** — это твой биллинг, а облачные ключи в чат совать нельзя. Но всё остальное
описано в коде; твоя часть — несколько кликов и пара секретов.

---

## Путь A — Render (рекомендую: без сервера, без Docker, без SSH)

Render сам читает `render.yaml` и поднимает всё.

1. Зайди на **https://render.com**, зарегься через **GitHub**.
2. **New + → Blueprint →** выбери репозиторий **`kisa134/bentlyk`**.
3. Render покажет переменные из `render.yaml`. Впиши минимум два:
   - `BENTLYK_LLM_API_KEY` — ключ WaveSpeed (`wsk_...`)
   - `TELEGRAM_BOT_TOKEN` — токен бота
   (по желанию: `BENTLYK_GH_TOKEN` — чтобы писал свой код; `TELEGRAM_ALLOWED_USER_ID`)
4. **Apply.** Render соберёт Dockerfile и запустит worker. Через пару минут — живой.

При каждом пуше в `main` он сам передеплоится. Логи — вкладка **Logs**. ~$7/мес
(у worker'ов нет бесплатного тарифа).

## Путь B — AWS EC2 (бесплатно первый год, возни больше)

Подробно — [`deploy-aws.md`](./deploy-aws.md). Коротко: создаёшь `t3.micro`
(Ubuntu 24.04), вставляешь заполненный [`scripts/aws-bootstrap.sh`](../scripts/aws-bootstrap.sh)
в поле **User data** — машина ставит всё сама + ночное автообновление.

## Проверить, что живёт (любой путь)

- **Дашборд** (Vercel): `https://<твой>.vercel.app/api/dashboard?key=...` —
  баннер «ЖИВ», тело покажет новый хост, сигналы дышат.
- **Telegram:** напиши боту. Скоро начнёт инициировать сам.

## Безопасность

Секреты, что светились в чате (бот-токен, WaveSpeed, GitHub PAT, пароль Supabase),
**ротируй** и держи только в env-переменных платформы — не в коде.
